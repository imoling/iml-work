# 搜索简历执行流程

## 1. 需求澄清与参数提取
- 接收用户输入的岗位名称、行业或具体招聘信息。
- 解析关键筛选条件：职位名称、工作经验要求、学历、地点、技能标签。
- 若用户输入信息模糊（如仅提供“找个工程师”），需引导用户提供具体行业（前端/后端/算法）或核心技术栈。

## 2. 执行搜索动作 (Playwright)
- 启动浏览器实例，访问指定的招聘平台或内部ATS系统（根据预设配置）。
- 在搜索栏输入提取到的关键词（如职位名称+核心技能）。
- 设置筛选条件：根据用户意图自动勾选或输入相应的经验年限、学历要求等。
- 执行搜索并提交表单。

## 3. 结果处理与匹配
- 等待页面加载完成，定位简历列表区域。
- 爬取前N条（默认10-20条）简历的关键信息：姓名、联系方式、现职公司、工作年限、核心技能。
- 进行初步匹配度打分：依据JD中的核心硬性指标（如必须有Java经验、5年以上经验）进行过滤。

## 4. 结果反馈
- 将筛选后的优质简历整理为结构化数据（JSON或列表格式）。
- 输出内容包括：
  - 匹配度最高的候选人名单。
  - 每位候选人的简要亮点（如“精通高并发”、“大厂背景”）。
  - 直接提供联系按钮或简历下载链接（视系统权限而定）。
- 询问用户是否需要查看某位候选人的详细简历或进行下一轮筛选。

## 5. 异常处理
- 若搜索无结果，提示用户放宽条件（如扩大搜索范围、减少硬性要求）。
- 若页面加载失败，重试一次；仍失败则告知用户“搜索服务暂时不可用，请稍后再试”。
